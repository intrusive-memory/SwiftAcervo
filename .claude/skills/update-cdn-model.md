# Update CDN Model

Upload or update a model on the Cloudflare R2 CDN used by SwiftAcervo.

## Context

SwiftAcervo downloads AI models exclusively from the private CDN at:
`https://pub-8e049ed02be340cbb18f921765fd24f3.r2.dev/models/{slug}/`

Each model has a `manifest.json` alongside its files that contains SHA-256 checksums for every file. The client verifies integrity against this manifest on every download.

## CDN Layout

```
models/
  {org}_{repo}/
    manifest.json          ← generated by Tools/generate-manifest.sh
    config.json
    tokenizer.json
    model.safetensors
    ...
```

## Workflow

When the user asks to upload or update a model:

1. **Confirm the model ID** (must be `org/repo` format, e.g., `mlx-community/Qwen2.5-7B-Instruct-4bit`)

2. **Ask which files** — all files, or specific ones?

3. **Check prerequisites**:
   ```bash
   which rclone && which huggingface-cli && echo "Tools available"
   ```
   If missing, guide installation:
   - `brew install rclone` then `rclone config` to add R2 remote named "r2"
   - `pip install huggingface-hub` for `huggingface-cli`

4. **Run the upload script**:
   ```bash
   ./Tools/upload-model.sh "org/repo" [file1 file2 ...]
   ```

   For gated models, set `HF_TOKEN`:
   ```bash
   HF_TOKEN=hf_xxx ./Tools/upload-model.sh "org/repo"
   ```

5. **If upload-model.sh is not available or you need manual control**, do it step by step:

   a. **Download from upstream source (HuggingFace)**:
   ```bash
   SLUG="org_repo"
   STAGING="/tmp/acervo-staging/$SLUG"
   mkdir -p "$STAGING"
   huggingface-cli download "org/repo" --local-dir "$STAGING" --local-dir-use-symlinks False
   rm -rf "$STAGING/.cache" "$STAGING/.huggingface"
   ```

   b. **Generate manifest**:
   ```bash
   ./Tools/generate-manifest.sh "org/repo" "$STAGING"
   ```

   c. **Upload to R2**:
   ```bash
   rclone copy "$STAGING" r2:pub-8e049ed02be340cbb18f921765fd24f3/models/$SLUG/ --progress
   ```

   d. **Verify**:
   ```bash
   curl -sI "https://pub-8e049ed02be340cbb18f921765fd24f3.r2.dev/models/$SLUG/manifest.json"
   ```

6. **Verify the manifest** by downloading and checking it:
   ```bash
   curl -s "https://pub-8e049ed02be340cbb18f921765fd24f3.r2.dev/models/$SLUG/manifest.json" | python3 -m json.tool
   ```

## Updating an Existing Model

When a model is updated upstream and needs to be re-synced:

1. Re-run the same upload process — it overwrites the existing files
2. The manifest is regenerated with new checksums
3. Existing clients will fetch the new manifest on their next download attempt
4. Files that match the old checksums will be re-downloaded (the manifest checksums changed)

## Environment Variables

| Variable | Default | Description |
|----------|---------|-------------|
| `RCLONE_REMOTE` | `r2` | rclone remote name |
| `R2_BUCKET` | `pub-8e049ed02be340cbb18f921765fd24f3` | R2 bucket name |
| `HF_TOKEN` | (none) | HuggingFace token for gated upstream models |
| `STAGING_DIR` | `/tmp/acervo-staging` | Local staging directory |

## Manifest Format

```json
{
  "manifestVersion": 1,
  "modelId": "org/repo",
  "slug": "org_repo",
  "updatedAt": "2026-03-22T00:00:00Z",
  "files": [
    {"path": "config.json", "sha256": "...", "sizeBytes": 1234},
    {"path": "model.safetensors", "sha256": "...", "sizeBytes": 4700000000}
  ],
  "manifestChecksum": "sha256-of-sorted-concatenated-file-checksums"
}
```

The `manifestChecksum` is computed by: sorting all file SHA-256 hashes lexicographically, concatenating them into one string, then computing the SHA-256 of that concatenation.
